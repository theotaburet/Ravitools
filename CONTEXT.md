# CONTEXT.md

## Routing

Utiliser cette arborescence comme architecture de contexte, inspiree d'ICM.

- `stages/01-discovery/`: comprendre l'existant, le besoin produit, les contraintes GPS/OSM.
- `stages/02-architecture/`: definir la cible technique et la repartition client/serveur.
- `stages/03-implementation/`: realiser les changements de code ou de structure.
- `stages/04-validation/`: verifier comportement, export, perf et decisions.

## Regle

Pour une nouvelle tache, commencer par le stage le plus bas niveau necessaire. Ne pas charger les autres stages sans raison.
